GitHub.io로 블로그를 만들려고 하니 GitHub에 커밋하기 전 블로그 모양새나 글을 확인하기 위해 로컬에 서버를 띄울 필요를 느꼈다. 로컬에 서버를 띄우지 않고 사용할 수도 있지만 매우 불편하고 제한적이다. 그렇게 할 거면 그냥 티스토리를 쓰겠음..

## 1. 원하는 테마 fork 해오기

깃허브에서 'jekyll theme' 등으로 검색을 하면 여러 개의 저장소가 나온다. 그 중 마음에 드는 테마를 fork 해서 내 깃허브로 가져온다. 나는 [이 테마](https://github.com/mmistakes/hpstr-jekyll-theme) 를 사용했다. 내가 사용한 테마의 경우 브랜치가 두 개가 있는데  이 중 gh-pages는 사용하지 않으니 제거해도 된다. fork 해올 때 저장소의 이름을 my-github-id.github.io로 설정하면 자동으로 웹페이지로 설정된다. 

## 2. 로컬에 소스코드 다운로드

Git Bash를 사용하던, 아니면 쉽고 편한 GitHub Desktop을 설치해서 사용하던, 원하는 폴더에 소스코드를 다운받으면 된다. 나는 GitHub Desktop의 기본 경로인 `C:\Users\wsong\Documents\GitHub\wsong0101.github.io`에 master 브랜치를 clone 했다. 

## 3. 윈도우에 Ubuntu 설치

Jekyll을 로컬에서 실행하려면 Ruby가 필요하다. 그런데 Ruby는 윈도우에서 설치가 매우 거지같다. 하지만 우리는 갓도우10의 은총으로 Ubuntu를 윈도우 스토어에서 무료로 설치할 수 있다. 쉽고 빠르다. Microsoft Store에서 'Ubuntu'로 검색하면 바로 나온다. 그냥 설치하고 실행해서 Id/Pw만 설정하면 끝.

## 4. 필요 프로그램 설치

아래 명령어들만 순서대로 쭉 실행해주면 (별 문제가 없을 경우) 그대로 서버가 뜰 수 있다. 먼저 루비를 설치한다.

```yaml
sudo apt-get update
sudo apt-get upgrade -y
sudo apt-get install ruby ruby-dev build-essential -y
```

그 다음은 아래 명령어로 환경변수를 설정해준다.

```yaml
echo '# Install Ruby Gems to ~/gems' >> ~/.bashrc
echo 'export GEM_HOME=$HOME/gems' >> ~/.bashrc
echo 'export PATH=$HOME/gems/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

그냥 순서대로 쭉 실행 해주자. 마지막으로 아래 명령어로 Jekyll을 설치한다.

```yaml
gem install jekyll bundler
```

다 되었으면 Ubuntu에서 쉽게 폴더에 접근하기 위해 링크 폴더를 만들어준다.

```yaml
ln -s /mnt/c/Users/wsong/Documents/GitHub/wsong0101.github.io ~/
```

보면 알겠지만 윈도우 Ubuntu에서는 /mnt/c/ 경로를 통해 c 드라이브에 접근할 수 있다. /mnt/c/ 이후의 경로는 본인이 clone 해온 폴더의 경로를 입력하자. 이렇게 하면 ~/ 경로에서 wsong0101.github.io로 바로 접근할 수 있다. 이 폴더 안으로 들어간다. 물론 여러분은 폴더명이 다를 것이다.

```yaml
cd wsong0101.github.io
```

## 5. 서버 실행

서버 실행에 앞서 이 프로젝트를 구성하는데 필요한 라이브러리들을 gem을 통해서 다운로드 받는다.

```yaml
bundle exec jekyll build
```

이 작업은 맨 처음 실행할 때 한 번만 하면 된다.
그리고 아래 명령어로 실제 서버를 실행한다.

```yaml
bundle exec jekyll serve
```

만약 잘 실행이 됐으면 아래와 같은 메시지가 보일 것이다.

![Smithsonian Image]({{ site.url }}/images/2018-11-07/success_result.png)

이제 브라우저에 들어가서 localhost:4000이라고 주소창에 쳐보자. 내가 fork 해온 그 블로그가 떠있으면 성공이다.

## 6. 블로그 수정
소스코드의 수정은 윈도우에서 에디터를 사용하여 할 수 있고 (나는 Visual Studio Code를 사용한다) 서버는 Ubuntu를 통해서 실행된다. 만약 블로그를 수정하고 그 결과를 편하게 확인하고 싶다면 서버를 띄울 때 아래와 같이 띄우면 된다.
```yaml
bundle exec jekyll serve --watch
```
--watch 옵션은 소스코드가 바뀌었을 경우 실시간으로 서버도 갱신하는 개발용 옵션이다. 에디터로 소스코드를 수정하고, 브라우저에서 새로고침을 하면 바로 변경 사항을 확인할 수 있으니 개발이 매우 편리해진다. 수정이 완료되면 GitHub에 커밋하는 것으로 배포 완료!
